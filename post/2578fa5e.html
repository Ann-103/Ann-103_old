Hardware Background

存储器分级的主要区别是访问速度，也是如今速度和容量不匹配的市场因素

Memory Hierarchy

名称 解释 速度 补充
寄存器 字为单位，存放操作数或地址 很快 几十到几百个，嵌入式可能只有几个
主存（内存） - CPU 控制部件只能从主存中取得指令和数据
- CPU 与外围设备交换信息依赖		 访远低于CPU执行指令速度 成本高，易失性
高速缓存 Cache 主存与 CPU 之间的一级存储器，静态存储芯片 SRAM
磁盘缓存 OS为磁盘IO在物理内存的区域 读写均要检查，写的时候可能需要更新
辅存（外存） 固定磁盘、可移动存储介质是OS设备管理的管辖范畴 较慢 便宜，非易失性。

Compilation, Linking and Loading of Programs 程序的编译、链接和装入

地址空间

  • 程序装载到内存才可以运行，一般保存在磁盘（.exe)
  • 多道程序设计模型，运行程序需建立进程，首先分配内存 ，允许多个程序同时进入内存
  • 每个进程有自己的地址空间

内存地址

  • 逻辑地址：相对，虚拟地址。编译、汇编后形成目标代码 .o为相对首地址编址

  • 物理地址：绝对，实地址。内存中存储单元的地址 可直接寻址

程序装入内存

装入方式

静态重定位：进程装入内存时，一次性完成到物理地址PA的转换，运行时不再进行，可以由软件完成

动态重定位：执行到对应语句时才完成到 PA 的转换，需要硬件支持（修改基址寄存器）

重定位：在装入时对目标程序中指令和数据的修改过程，

  • 编译程序产生绝对地址的目标代码，也可由程序员直接赋予。通常程序员宁可在程序中采用符号地址，然后在
    编译或汇编时，再将这些符号地址转换为绝对地址。 （变量）

  • 装入内存后不需对程序和数据的地址进行修改

多道程序环境下，不可能预知目标模块放在内存中的地址，因此绝对装入方式不适合在多道环境下使用

特征 静态重定位 动态重定位
可执行文件中的地址 相对地址 相对地址
装入内存的地址 绝对地址 相对地址
进程在运行时可以在内存中移动：Swap / Compacting 否 是
进程的内存空间必须连续 是 是
硬件支持 否 MMU

动态重定位硬件支持

基址寄存器和限长寄存器 ：为了地址转换不会影响到指令执行速度的必须有硬件地址变换机构

  • 程序从内存的某处移动到另一处时，只需修改基址寄存器的值即可
  • 物理地址 = 逻辑地址 + 基址寄存器中的地址值
方式 介绍 补充
绝对装入方式 仅适用于单进程系统 ， 编译时完成到 PA 的转换
静态重定位 进程装入时完成到物理地址的转换，运行时不再进行

Real Memory Management

存储器管理的功能

说明 解释 补充
内存分配 把空闲的主存区分配给作业或进程 最大化利用内存，切分、分配以便将尽可能多的进程装入内存
地址重定位 /地址映射
relocation		 运行时将虚拟地址转换成物理地址的过程 硬件和OS软件必须能把程序中的内存访问转换为实际的物理内存地址，并反映程序在内存中的当前位置
进程保护 用户进程不能访问os程序和数据，进程之间不能跳转指令或者数据区 **处理器（硬件）**满足，因为os不能预测检查，开销大
进程共享 保护机制允许多个进程访问内存的同一部分
虚拟存储 用户程序的大小和结构不受主存容量和结构的限制，即使大于主存也可运行
逻辑组织* 内存和外存总是被组织成线性的地址空间 OS和硬件能处理模块形式组织的用户程序与数据可以独立编译编写模块，提供不同保护级别，实现共享等（分段）
物理组织* 系统主要关注的是内存和外存之间信息流的组织

Memory Partitioning 内存分配

分区是指内存中的一个连续区域。

概要

  • 连续（分区管理方式）：连续的内存空间
    • 固定（定长或静态分区分配）：Fixed partition: shelving the boxes 最简单的可运行多道程序的内存管理方式
      • Equal-size partitions：过大浪费，过小不够
      • Unequal-size partitions ：根据程序大小决定所使用的分区
    • 动态: Dynamic Partition: stacking the boxes
    • 伙伴：Buddy System: splitting & merging the shelves
    • 单一连续分配：单用户、单任务 OS，一个程序一个连续内存空间
  • 非连续：一个进程分散地装入到不相邻接的分区
    • 页式管理
    • 段式管理

分区管理方式

内存分为系统区和用户区，一般情况下无存储器保护机构

  • 系统区：仅提供给OS使用，通常是放在内存的低址部分；
  • 用户区：除系统区以外的全部内存空间，提供给用户使用

特点：简单易行，系统开销小，资源利用率低(一次只能装入一个作业)

描述：静态地把内存用户空间划分为若干个固定大小的连续区域，每个分区中只装入一道进程

内存分配表(分区说明表)：分区号、分区大小、起始地址和分区的使用状态。内存的分配释放、存储保护以及地址变换都通过分区说明表进行

进程表：进程名、进程大小、进程占用的分区号

过程： 内存分配程序检索表分配或拒绝

  • Equal-size partitions ：无特殊，每个分区都需要维护一个调度队列
  • Unequal-size partitions ：
    • per-partition queue ：减少内部碎片，单个分区最优，等待，小内存的进程可能被阻塞即使有大的空闲分区
    • global queue: 所有的进程只提供一个队列，有都被占据，则必须进行交换，换出阻塞进程等

优缺点：

  • 优：进程在执行时，不会改变存储区域，所以采用静态地址重定位方式。此方法易于实现，系统开销小
  • 缺：分区的数量在系统生成阶段已经确定，因而 限制了系统中并发进程的数量
  • 缺：分区的大小是在系统生成阶段实现设置的，因而 小作业不能有效地利用分区空间
  • 分区长度和数量是可变的，动态分区方法会在内存中形成许多小空洞（外部碎片），内存利用率随之下降。
  • 克服外部碎片的一种方法是压缩，非常耗时且需要动态重定位的能力
概念 解释 补充
内部碎片 一个存储分区内
外部碎片 通常指一个暂时不能使用的存储分区
分配分区表 描述已被分配的区域
空闲分区表 描述内存中的自由区域 一般的实现是一条空闲分区链
空闲分区链 为每一个自由分区设置一个链接指针来指向下一个自由分区，使所有的自由分区形成一个链表。 含头指针

First-fit > Next-fit > Best-fit

放置算法 描述 补充 链表顺序
First-fit 从头开始扫描，选择大小足够的第一个可用块 最简单有效，低地址小分区多，高地址预留大分区，开销大 地址递增
Next-fit 从上一次放置的位置开始，选择下一个可用块 空闲分区均匀，减少开销，但是缺乏大的分区 地址递增循环
Best-fit 能满足要求、又是最小的分区 性能较差，小分区多，解决大进程分配 分区递增
Worst-fit 检查第一个是否可用 小空间少，大空间不会大，中小型作业 分区递减

比较

  • 最优适应分配算法的主存利用率最好 ，但某些情况下，最坏更好。
  • 如果空闲区按从小到大排列，则最先适应分配算法等于最优适应分配算法，反之等于最坏。
    • 这里是因为最先适应按照地址链接，最优按照分区大小，如果分区大小递增，相当于从低地址链接到高地址。

Paging 分页

晚于分段的出现，但普遍应用，因为简单且空块少

虚拟地址的空间：电脑位数，2^32 字节，物理地址空间要比虚拟地址小

分页产生地址结构，页面编号从0开始，页的数据是1024bit。这样可以直接通过移位操作做除法，高位是页号，从0开始，低位是偏移量

地址存储的K是2的多少次方。

缺页，无对应叶框，MMU产生中断信号，有中断向量，将操作系统唤醒，查找中断向量表，block进程，进行io，找硬盘，查找程序完成后再次中断唤醒os，os将进程放入ready，然后进行调度

缺页中断和io中断导致内核态和用户态切换，由寄存器到内存到硬盘的速度降低

本身我们从ptbr找页表就多做了一次查找

改进

  • 减少内存访问，将已经查过的表项先保存，先在寄存器组中查看，此时页表并不是一个连续的状态，利用寄存器本身的小处理器实现并行，同时查看（硬件处理），即TLB（快表或联想寄存器组）。贵且耗电（散热），故小规模，更新策略如果是先进先出，串行程序很好，但是如果是循环则over，另外策略是存一个符号表示最近访问

    期望百分之九十在这个cache，百分之九十九并不用查页表（页表在内存里，还要再访问内存数据，一共查两次）

分页的共享与保护

倒排页表

  • 区别于传统页表面向进程空间（逻辑地址空间），进程逻辑页面有一个表项
  • 倒排页表面向内存空间(物理地址空间) ，每个内存页框一个表项，大小固定
    ❖ 整个系统共用一个页表

缺省页表，需要寄存器组（中断），调查对应页的权限。观察页表是否满，满则替换，寄存器恢复，重置pc继续跑

页表项中有一个位用于标志是否修改的位，在被踢出去之前，先把修改内容写出去，确保这一页的修改不会丢失

缺页中断的处理

  • 发起缺页中断：MMU 查询页表失败，即目标内存页已被 swap 出内存
  • 中断唤醒OS
  • OS 将进程置为 Blocked
  • OS 发起 I/O 请求，将页面从磁盘中 swap 入内存
  • 在 I/O 操作在后台运行的时候，CPU 将让位给下一个进程
  • I/O 完成，硬件发起 I/O 中断，OS 返回
    • 修改页表
    • 将上述进程重新置为 Ready

出现原因：并没有将所有页表的索引都放入MMU

Segmentation 分段

概述：

  • 碎片多，不好管理，数据也比较混乱（动态分配，堆栈对空间的利用程度取决于动态分配 ）

优点

  • 简化对不断增长的数据结构处理
  • 允许程序独立改变或重新编译
  • 进程共享
  • 保护

信息分享与保护

FIFO：访问不算在内，而是放入和替换时间，可能0很早之前就进来了，但是会经常访问，此时如果满了的话还是会踢出去0

LRU：频率一致比较时间
