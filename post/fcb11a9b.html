实验 0： 背景介绍

参考：https://th0ar.gitbooks.io/xv6-chinese/content/content/chapter0.html

Xv6操作系统

XV6是MIT为了os的课程开发的一个教学目的的操作系统，在x86处理器上使用c实现的unix v6，使用的 是ANSI C 发布之前的旧式 c语言。提供Unix 操作系统中的基本接口，模仿内部设计。

Xv6操作系统接口

内核

xv6 提供了传统的内核概念

内核kernel是os的基本部分， 为众多应用程序提供对计算机硬件的安全访问的一部分软件，这种访问是有限的，并且内核决定一个程序在什么时候对某部分硬件操作多长时间。

  • 进程通过系统调用使用内核服务。系统调用会进入内核，让内核执行服务然后返回。所以进程总是在用户空间和内核空间之间交替运行。

  • 系统调用 system call

    system call 描述 system call 描述
    fork() 创建进程 exit() 结束当前进程
    wait() 等待子进程结束 sleep(n) 睡眠n秒
    kill(pid) 结束pid所指进程 exec(filename,*argv) 加载并执行一个文件
    getpid() 获取当前进程pid sbrk(n) 为进程内存空间增加n字节
    open(filename,flags) 打开文件，flag指定读写模式 close(fd) 关闭打开的fd
    read(fd,buf,n) 从文件中读取n个字节到buf dup(fd) 复制fd
    write(fd,buf,n) 从buf中写n个字节到文件 pipe§ 创建管道，并把读和写的fd返回到p
    chdir(dirname) 改变当前目录 fstat(fd) 返回文件信息
    mkdir(dirname) 创建新的目录 link(f1,f2) 给f1创建一个新名字（f2）
    mknod(name,major,minor) 创建设备文件 unlink(filename) 删除文件

shell

接受用户的命令并执行，相当于一个cmd，并不是特别的用户程序，或者内核的一部分，只是一个shell而已

进程

xv6无TCB结构，即无进程抽象，使用proc的结构表示线程，都有自己的context

  • 进程有自己的指令（程序运算）、数据(变量)、栈（管理程序的内存调用）的内存空间
  • 一个xv6进程由两部分组成，一部分是用户内存空间（指令、数据、栈），另一部分是仅对内核可见的进程状态。
  • 分时特性：进程在可用 CPU 之间不断切换，当一个进程不在执行时，xv6 保存它的 CPU 寄存器，当他们再次被执行时恢复这些寄存器的值。内核将每个进程和一个 pid 关联。
  • **proc**结构类似于Linux中的task struct，用于实现线程，或者说进程结构的复用
  • 系统调用：进程（程序）进行系统调用，系统调用进入内核执行命令
    • 进程可以通过系统调用 fork 来创建一个新的进程，父进程返回子进程pid，子进程不返回
    • exit 会导致调用它的进程停止运行，并且释放诸如内存和打开文件在内的资源。
    • wait 会返回一个当前进程已退出的子进程，如果没有子进程退出，wait 会等候直到有一个子进程退出。
    • 用 exec 将从某个文件（通常是可执行文件）里读取内存镜像，并将其替换到调用它的进程的内存空间

文件描述符

常称为“文件”，一个整数，（看上去像是字节流？），它代表了一个进程可以读写的被内核管理的对象，进程可以通过多种方法获得一个文件描述符，如打开文件、目录、设备或者创建一个管道、或者复制已经存在的文件描述符。

  • 每个进程都有一个从0开始的文件描述符空间 ：xv6将文件描述符作为进程表的索引

  • 系统调用

    • read和write从文件描述符所指的文件中读或者写n个字节。read从fd中读最多n个字节，将它们拷贝到buffer中，然后返回读出的字节数。每一个指向文件的文件描述符都和一个偏移量有关。

    • linux中一切皆文件，分普通和目录文件等，目录文件包含了哥哥文件和执行文件的指针

    • 任何硬件设备都必须跟根目录下某一目录执行挂载让文件系统在os中可用，mount命令操作，否则无法使用

    • cat的实现：将数据从标准输入复制到标准输出，自己不知道它是从文件、控制台或者管道中读取或者写数据的

    • close 会释放一个文件描述符，使得它未来可以被 open, pipe, dup 等调用重用

    • 一个新分配的文件描述符永远都是当前进程的最小的未被使用的文件描述符。

  • I/O重定向实现：将一个文件描述符标识的文件替换为另一个文件

    • fork 会复制父进程的文件描述符和内存
    • exec 会替换调用它的进程的内存但是会保留它的文件描述符表。
    • 单独调用的原因： shell 可以在子进程执行指定程序之前对子进程进行修改。

    举例： 在shell中输入指令cat<input.txt，shell会执行代码

    • 子进程关闭文件描述符0后
    • open 会使用0作为新打开的文件 input.txt的文件描述符（因为0是 open 执行时的最小可用文件描述符）
    • cat 就会在标准输入指向 input.txt 的情况下运行，实现新建文件的功能

页表 pagetable

xv6使用页表（由硬件实现）来为每个进程提供其独有的地址空间。

将虚拟地址翻译为物理地址。xv6为每个进程维护了不同的页表，这样就能够合理地定义进程的地址空间了，一片地址空间包含了从虚拟地址0开始的用户内存。

还能够为不同的进程的内存提供保护
