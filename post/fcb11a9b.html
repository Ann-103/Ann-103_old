实验 0： 背景介绍

参考：https://th0ar.gitbooks.io/xv6-chinese/content/content/chapter0.html

Xv6操作系统

XV6是MIT为了os的课程开发的一个教学目的的操作系统，在x86处理器上使用c实现的unix v6，使用的 是ANSI C 发布之前的旧式 c语言。提供Unix 操作系统中的基本接口，模仿内部设计。

实验拉取：

git clone git://g.csail.mit.edu/xv6-labs-2020

安装包

sudo apt-get install git build-essential gdb-multiarch qemu-system-misc gccriscv64-linux-gnu binutils-riscv64-linux-gnu

Xv6操作系统接口

内核

xv6 提供了传统的内核概念

内核kernel是os的基本部分， 为众多应用程序提供对计算机硬件的安全访问的一部分软件，这种访问是有限的，并且内核决定一个程序在什么时候对某部分硬件操作多长时间。

  • 进程通过系统调用使用内核服务。系统调用会进入内核，让内核执行服务然后返回。所以进程总是在用户空间和内核空间之间交替运行。

  • 系统调用 system call ：进程（程序）进行系统调用，系统调用进入内核执行命令

    system call 描述 system call 描述
    fork() 创建进程 exit() 结束当前进程
    wait() 等待子进程结束 sleep(n) 睡眠n秒
    kill(pid) 结束pid所指进程 exec(filename,*argv) 加载并执行一个文件
    getpid() 获取当前进程pid sbrk(n) 为进程内存空间增加n字节
    open(filename,flags) 打开文件，flag指定读写模式 close(fd) 关闭打开的fd
    read(fd,buf,n) 从文件中读取n个字节到buf dup(fd) 复制fd
    write(fd,buf,n) 从buf中写n个字节到文件 pipe§ 创建管道，并把读和写的fd返回到p
    chdir(dirname) 改变当前目录 fstat(fd) 返回文件信息
    mkdir(dirname) 创建新的目录 link(f1,f2) 给f1创建一个新名字（f2）
    mknod(name,major,minor) 创建设备文件 unlink(filename) 删除文件

  • exit 会导致调用它的进程停止运行，并且释放诸如内存和打开文件在内的资源。

  • wait 会返回一个当前进程已退出的子进程，如果没有子进程退出，wait 会等候直到有一个子进程退出。

  • 用 exec 将从某个文件（通常是可执行文件）里读取内存镜像，并将其替换到调用它的进程的内存空间

shell

接受用户的命令并执行，相当于一个cmd，并不是特别的用户程序，或者内核的一部分，只是一个shell而已

在shell中我们调用system call，system call 进入内核调用并执行对应代码，这个代码我们可以自己在user 文件夹中编写，并在makefile 文件中包含路径。

进程

结构体

xv6无TCB结构，即无进程抽象，使用proc的结构表示线程，最重要的是：页表（内存管理），内核栈(kstack)，当前运行状态（仅内核可见）。

这里不用上下文切换，而是运行线程（线程）来执行进程的指令，可以被挂起或运行，来表示进程的切换。

线程的大多数状态（局部变量和函数调用的返回地址）都保存在线程的栈上

进程的分时特性：进程在可用 CPU 之间不断切换，当一个进程不在执行时，xv6 保存它的 CPU 寄存器，当他们再次被执行时恢复这些寄存器的值。内核将每个进程和一个 pid 关联。

用户栈和内核栈（p->kstack）

  • 进程运行用户指令时，只有其用户栈被使用，其内核栈则是空的

  • 当进程（通过系统调用或中断）进入内核时，内核代码就在进程的内核栈中执行，用户栈仍然保存着数据，只是暂时处于不活跃状态

    处理器转入内核栈中，提升硬件的特权级，然后运行系统调用对应的内核代码。当系统调用完成时，又从内核空间回到用户空间：降低硬件特权级，转入用户栈，恢复执行系统调用指令后面的那条用户指令。线程可以在内核中“阻塞”，等待 I/O, 在 I/O 结束后再恢复运行。

  • 进程的线程交替地使用着用户栈和内核栈。内核栈是用户代码无法使用的，这样即使一个进程破坏了自己的用户栈，内核也能保持运行。

状态

p->state 指示了进程的状态：新建、准备运行、运行、等待 I/O 或退出状态中。

页表

xv6使用页表（由硬件实现）来为每个进程提供其独有的地址空间。

为每个进程维护了不同的页表，合理地定义进程的地址空间了

p->pgdir 以 x86 硬件要求的格式保存了进程的页表。xv6 让分页硬件在进程运行时使用 p->pgdir。进程的页表还记录了保存进程内存的物理页的地址。

  • 一个xv6进程由两部分组成，一部分是用户内存空间（指令、数据、栈），另一部分是仅对内核可见的进程状态。（内核的数据映射）？？

一片地址空间包含了从虚拟地址0开始的用户内存。它的地址最低处放置进程的指令，接下来则是全局变量，栈区，以及一个用户可按需拓展的**“堆”**区（malloc 用）

将虚拟地址翻译为物理地址

  • 内核的指令和数据也会被进程映射到每个进程的地址空间中，进程使用系统调用时，系统调用实际上会在进程地址空间中的内核区域执行，可以让内核的系统调用代码可以直接指向用户内存
  • xv6 将内核映射到了地址空间的高地址处，即从 0x80100000 开始（给用户留内存）

文件描述符

文件描述符是一个强大的抽象，将连接的细节隐藏：一个进程向描述符1写出，它有可能是写到一份文件，一个设备（如控制台），或一个管道。

打开现存文件或新建文件时，内核会返回一个文件描述符。读写文件也需要使用文件描述符来指定待读写的文件

  • 0: 标准输入、1: 标准输出、2: 标准错误

常称为“文件”，一个整数，（看上去像是字节流？），它代表了一个进程可以读写的被内核管理的对象，进程可以通过多种方法获得一个文件描述符，如打开文件、目录、设备或者创建一个管道、或者复制已经存在的文件描述符。

  • 每个进程都有一个从0开始的文件描述符空间 ：xv6将文件描述符作为进程表的索引

  • 系统调用

    • read和write从文件描述符所指的文件中读或者写n个字节。read从fd中读最多n个字节，将它们拷贝到buffer中，然后返回读出的字节数。每一个指向文件的文件描述符都和一个偏移量有关。

    • linux中一切皆文件，分普通和目录文件等，目录文件包含了哥哥文件和执行文件的指针

    • 任何硬件设备都必须跟根目录下某一目录执行挂载让文件系统在os中可用，mount命令操作，否则无法使用

    • cat的实现：将数据从标准输入复制到标准输出，自己并不知道两个具体对象

    • close 会释放一个文件描述符，使得它未来可以被 open, pipe, dup 等调用重用

    • 一个新分配的文件描述符永远都是当前进程的最小的未被使用的文件描述符。

  • I/O重定向实现：将一个文件描述符标识的文件替换为另一个文件

    • fork 会复制父进程的文件描述符和内存
    • exec 会替换调用它的进程的内存但是会保留它的文件描述符表。
    • 单独调用的原因： shell 可以在子进程执行指定程序之前对子进程进行修改。

    举例： 在shell中输入指令cat<input.txt，shell会执行代码

    • 子进程关闭文件描述符0后
    • open 会使用0作为新打开的文件 input.txt的文件描述符（因为0是 open 执行时的最小可用文件描述符）
    • cat 就会在标准输入指向 input.txt 的情况下运行，实现新建文件的功能

  • 父子进程与文件描述符

    • fork 复制文件描述符，但是每个文件的偏移共享

    • dup 复制一个已有的文件描述符，返回一个指向同一个输入/输出对象的新描述符，同样共享文件进度

管道

  • 管道是一个小的内核缓冲区，它以文件描述符对的形式提供给进程，一个用于写操作，一个用于读操作。

  • 从管道的一端写的数据可以从管道的另一端读取。管道提供了一种进程间交互的方式。

文件系统

xv6 关于文件系统的操作都被实现为用户程序，诸如 mkdir，ln，rm 等等。这种设计允许任何人都可以通过用户命令拓展 shell

例外：cd ，不能作为普通命令执行

Unix 时代的其他系统的设计都将这样的命令内置在了 shell 中，而 shell 又是内置在内核中的。

如果 cd 作为一个普通命令执行，那么 shell 就会 fork 一个子进程，而子进程会运行 cd，cd 只会改变子进程的当前工作目录。父进程的工作目录保持原样。

文件与目录

  • 文件就是一个简单的字节数组

  • 目录包含指向文件和其他目录的引用，xv6 把目录实现为一种特殊的文件。目录是一棵树，它的根节点是一个特殊的目录

  • /a/b/c 指向一个在目录 b 中的文件 c，而 b 本身又是在目录 a 中的，a 又是处在 root 目录下的

  • 不从 / 开始的目录表示的是相对调用进程当前目录的目录

  • chdir调用进程的当前目录

  • 打开同一个文件

  • open 加上 O_CREATE 标志打开一个新的文件

  • mknod 创建一个新的设备文件,但是这个文件没有任何内容。

qemu

一个虚拟机，对于单片机或者嵌入式常用的处理器模拟。

主要提供两种功能给用户使用。

  • 是作为用户态模拟器，利用动态代码翻译机制来执行不同于主机架构的代码。
  • 是作为虚拟机监管器，模拟全系统，利用其他VMM(Xen, KVM, etc)来使用硬件提供的虚拟化支持，创建接近于主机性能的虚拟机。
