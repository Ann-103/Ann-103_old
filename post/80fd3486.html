环境配置

发表于2023-04-11|更新于2023-07-02|其他
|字数总计:961|阅读时长:3分钟|阅读量:

主要是解决一些小白时期电脑环境配置的问题，另外配置一下深度学习的环境。

Python

一些介绍：

Anaconda就是可以便捷获取包且对包能够进行管理，同时对环境可以统一管理的发行版本。Anaconda包含了conda、Python、numpy在内的超过180个科学包及其依赖项。

conda是包及其依赖项和环境的管理工具， 不会影响系统自带python。

pip是用于安装和管理软件包的包管理器。pip3 install

pip可以允许你在任何环境中安装python包，而conda允许你在conda环境中安装任何语言包

重装Anaconda

  • add anaconda to the system path 不勾选则仅在运行才会将anaconda安装的py添加到环境变量，不影响电脑原本的py，在cmd中查看py版本仍旧是原本的。

    但是我安装的最新版本好像并不是这个界面了，直接默认安装

  • 添加path

    D:\Anaconda
    D:\Anaconda\Scripts
    D:\Anaconda\Library\mingw-w64\bin
    D:\Anaconda\Library\bin

  • cmd查看安装版本

  • cmd更改镜像源头

  • cmd中使用conda

  • conda环境管理

  • conda 包管理

    • conda install xxx：这种方式安装的库都会放在anaconda3/pkgs目录下

      好处：conda不同环境中需要库可以直接复制，不会重复安装

    • pip install xxx：如果当前conda环境的python是conda安装的，和系统的不一样，那么xxx会被安装到anaconda3/envs/current_env/lib/python3.x/site-packages文件夹中，如果当前conda环境用的是系统的python，被安装到~/.local/lib/python3.x/site-packages文件夹中

    • 一般来说cmd中pip3也会直接安装到系统package中吧

  • conda激活环境后，直接运行py文件或者使用自带的idle运行简单的文件

  • pycharm：每创建一个项目都需要指定相应的Python解释器

    • 创建的时候选择环境
    • 修改：file-setting-Project Interpretor-齿轮-conda environment-existing environment-浏览安装包中env的对应文件夹下的环境

  • 内置jupyter notebook的网页交互：new新页面就可以实时交互了

    安利一下everything，检索文件极其方便

  • 不同位的python
