Chapter 3 语法分析

接受词法分析的内容并传递给下一个部分

上下文无关文法：另一种表示语言的方法

正规式表示：重复必须要指定的有限，或者无限。不能描述配对或嵌套

上下文无关文法（文法）：比正规式更加强，一个四元组（VT,VN,S,P）（V_T , V_N , S, P）（VT​,VN​,S,P）

  • VTV_TVT​ : 终结符集合 (即和待分析的串即代码的具体内容)

    • 字母表前面的小写字母，如a, b, c, …
    • 词法分析中约定的记号，如id, num, if, while, …
    • 数字0, 1, 2, … , 9
    • 标点符号，如逗号，句号，…
    • 运算符号，如+, -, *, <, >, =, …

  • VNV_NVN​: 非终结符集合

    • 字母表前面的大写字母，如A, B, C, …
    • 字母S，通常代表开始符号
    • 小写字母组成的名字，如expr, op, stmt, …

  • S : 开始符号，非终结符中

  • P : 产生式集合， 产生式形式 : A → α

  • 小写希腊字母，如α, β, γ, …，代表文法符号串

文法本身是不包含 “|” “*”等运算符的，将上面的内容进行简化表示：

  • 利用 | 表示产生式中可推导的不同结果比如 expr → expr op expr | (expr) | - expr | id
  • *表示乘法
  • 利用 ⇒\Rightarrow⇒* 表示零步或多步推导
  • 利用 ⇒\Rightarrow⇒+表示一步或多步推导

推导：把产生式（P）看成重写（替代）规则，把符号串中的非终结符用其产生式右部的串（可能是终结or非终结符）来代替

  • 最左推导：每一次都代换句型中最左边的非终结符
  • 最右推导（规范推导）
  • 分析树（本身不区分是最左还是最右推导出来的）

范例： E → E + E | E *E | (E ) | -E | id

  • E→E+EE\to E+EE→E+E表示两表达式相加仍然是表达式，允许使用右边的式子代替左边的式子。
  • E⇒E+EE \Rightarrow E+EE⇒E+E 表示“E推导出E+E”
  • 从开始符号使用产生式可以得到一个代换序列 E⇒E+E⇒id+E⇒id+idE\Rightarrow E+E\Rightarrow id+E\Rightarrow id+idE⇒E+E⇒id+E⇒id+id，表明了最后的结果是表达式的实例，所有项都是语法中的句型

二义性：未规定顺序，另外根据替换的顺序可以写成树

语言和文法

文法的优点

  • 精确，易理解的语法说明

  • 自动产生高效的分析器

  • 语言定义出层次结构

  • 便于修改

文法的问题

  • 文法只能描述编程语言的大部分语法，不能描述语言中上下文有关的语法特征

文法比正规式更强，但是在词法分析阶段还是使用正规式，词法规则非常简单，不必用文法，对于词法记号，正规式描述简洁且易于理解。从正规式构造出的词法分析器效率高

正规式和上下文无关文法的比较

  • 正规式描述的都可以用用上下文无关文法**描述

比如：正规式(a|b)*ab 构造的文法：

A0 → aA0 | bA0 | aA1
A1 → bA2
A2 → ε

避免回溯：匹配成功（暂时）即正确，或者要么就无法匹配成功

  • 第一个L代表从左向右扫描输入符号串；

定义函数（非终结符->终结符）：

  • FIRST(α) = {a | α →\to→ *a…, a∈VT}特别当 α →\to→ε 时，规定 ε ∈ FIRST(α) 即零步或多步推导到目标的集合。如果函数自变量是很多的串，则考虑推到的串的第一个字符。

  • FOLLOW(A) = {a | S →\to→* …Aa…， a∈VT}如果A是某个句型的最右符号，那么$属于FOLLOW(A) ，即开始符号推导出的在非终结符A后面的所有终结符

计算规则：

  • 第二个 B的开始符号一定是A的开始符号的子集，因为B能推导的首字符，A一定能推导，但是反之不一定。

  • 第二个 B如果可以推导到空串，则后面的α 开始符号也是A的子集

  • 最后一个

    • β的开始符号，一定是A的后继。

    • 后面对于$\beta $推导到空串，那么B的后继一定是A的后继，A可以推导出别的后继。

LL（1）文法分析：对于任意的 A → α | β | γ，（即每个产生式的分支都可以进行唯一可逆推的选择）对于输入a，有且仅有一种 first（α）= a 。

  • 所有的first 集合都没有空串，则只需 所有的开始集合不相交即可：FIRST(α) ∩ FIRST(β) = Ø ，可使用左因子提取的方法做到这一点
  • 如果存在某个非终结符能推导到 ε ， β $\implies $* ε，则即使满足上面的条件仍然有理由选择β，故观察下一个元素，也就是A的后继，那么需要保证 FIRST(α) ∩ FOLLOW(A) = Ø

对于产生式中多个选择，所有选择的开始符号都不相同，并且也不和产生式左侧符号的后继相同。

性质: 无公共左因子；不是二义的；不含左递归 。

对于文法中满足上面这两条的，可以称为LL（1）文法，我们也可以构建出非递归的算法进行输入串的分析（构造分析树）。

无普通适用的方法

预测分析的错误恢复

(1) 先对编译器的错误处理做一个概述

  • 词法错误，如标识符、关键字或算符的拼写错
  • 语法错误，如算术表达式的括号不配对
  • 语义错误，如算符作用于不相容的运算对象
  • 逻辑错误，如无穷的递归调用

(2) 分析器对错误处理的基本目标

  • 清楚而准确地报告错误的出现，并尽量少出现伪错误
  • 迅速地从每个错误中恢复过来，以便诊断后面的错误
  • 它不应该使正确程序的处理速度降低太多

(3) 非递归预测分析在什么场合下发现错误

  • 栈顶是终结符，但和输入符号不匹配
  • 栈顶是非终结符A，输入符号是a，但M[A , a]是空白

(4) 非递归预测分析：采用紧急方式的错误恢复

  • 发现错误时，分析器每次抛弃一个输入记号，直到输入记号
    属于某个指定的同步记号集合为止

(5) 同步

  • 词法分析器当前提供的记号流能够构成的语法构造，正是语
    法分析器所期望的
  • 不同步的例子
    • 语法分析器期望剩余的前缀构成过程调用语句，而实际剩余的前缀
      形成赋值语句

(6) 同步记号集合的选择

  • 把**FOLLOW(A)**的所有终结符放入非终结符A的同步记号集合
    • 例如： if expr then stmt
    • then和分号等记号是expr的同步记号
  • 把高层构造的开始符号加到低层构造的同步记号集合中
    • 例如： a = expr; if …
    • 语句的开始符号作为表达式的同步记号，以免表达式出错又遗漏分
    号时忽略 if 语句等一大段程序
  • 把FIRST(A)的终结符加入A的同步记号集合
    • 例如： a = expr; , if …
    • 语句的开始符号作为语句的同步符号，以免多出一个逗号时会把 if
    语句忽略了
  • 如果终结符在栈顶而不能匹配，弹出此终结符
  • 如果非终结符在栈顶且可以产生空串，则可以使用推出空串
    的产生式

类型化语言和类型系统

先来一大堆概念

（一）类型化的语言 如 c

  • 数据、变量、运算对象和结果都被给定类型 ，用来限制变量的取值范围

  • 可静态确定和检查：表达式、语句 等程序构造的类型

    显式类型化语言 ：类型是语法的一部分

    隐式类型化的语言 ：主流语言不存在，可能存在忽略类型信息的程序片段 ，如函数的参数类型不用声明

  • 类型检查：一般静态检查，但是也有可能需要动态的比如数组越界，可以全部动态但是影响效率

类型化语言，静态检查较早发现问题，类型信息有文档作用（比如结构体），程序模块可以相互独立运行，有效的空间安排

（二）未类型化语言 如LISP语言

  • 不限制变量值范围的语言
  • 结果可能是一个有意义的值、一个错误、一个异常或一个语言未加定义的结果
  • 类型检查：可以通过彻底的运行时详细检查（动态）排除all禁止错误

（三）程序运行时的执行错误

  • 会被捕获的错误 ：引起计算立即停止

    • 非法指令 、非法内存访问、除数为零

  • 不会被捕获的错误 ：可能会有一段时间未引起注意

    • 下标越界、错误地址（但瞎猫死耗子）

禁止错误

禁止错误：不会被捕获错误集合 + 会被捕获错误的一个子集，语言设计类型系统的目标是在排除禁止错误 ，上面两个也可以基于禁止错误实现

（四）类型系统：给定类型和检查

  • 语言的组成部分,它由一组定型规则（typing rule）构成 ，用于给程序构造指派类型

  • 目的：通常静态检查的方式来保证合法程序运行时的良行为 （无下标越界等）

    偶尔可能有动态

  • 定型规则 ：可以看成条件结果的推理规则，前提为空表示公理
    • line1：开始环境为空，即良性
    • line2：如果空环境是对，则添加新的表达式比如bool，int
    • line3：定义了规则，对表达式进行赋予类型
    • 通过环境规则判断环境断言正确，然后进行语法断言（表达式或者基础类型），后面有例子

类型检查

下面是一个语言的定义 ，使用抽象方式对类型变量进行描述

  • 基础类型
  • 构造类型：数组，函数等
  • 箭头表示指针

对应的类型系统规则：从环境规则到语法规则到定型规则

语法规则表示在良性环境下，任何的基础类型是类型表达式，同时也定义数组等构造类型

  • 编程语言和定型断言的类型表达式并非完全一致，分别属于语言系统和类型系统
  • void在文法中未出现，添加用来表示语句，本质上表示的是语句 没有类型
  • 约定写指针和数组的类型等，所有的数组下标 从0开始
  • 函数定义形参和返回值

对表达式和语句指派类型：最重要

  • 对应的就是定型断言
  • Exp Id语句：如果id出现在环境中，则id的类型就是环境中的类型，即查符号表找类型
  • 函数语句：表示E1是函数类型，E2是实参，那么函数访问返回值是T2类型
  • void表示有效语句，分别定义了语句是否有效，最后一行是串行语句是有效的

类型检查器

  • 依据上面的类型系统构造翻译方案，添加语义动作，构造类型表达式，并写入T的综合属性
  • 类型环境信息写入符号表，很多语言要求函数等先声明，就是要写入符号表
  • 类型表达式使用抽象语法
  • 增加报错类型 type_error，这也是类型检查器和类型系统的区别，后者理想无错，前者需要应对出错情况

多态函数

多态函数

  • 允许变元的类型有多种不同的情况
  • 函数体中的语句的执行能适应变元类型有多种不同的情况（区别于重载的特征）

多态算符

  • 即符号有多种效果比如 & 求地址或者与
  • 相应操作的代码段接受不同类型的数组、函数等
  • 例如：数组索引、函数应用、通过指针间接访问。 C语言手册中关于取地址算符&的论述是：“如果运算对象的类型是’…‘，那么结果类型是指向’…'的指针”

**类型变量 **

  • 便于讨论未知类型：不要求标识符提前声明，通过类型变量的使用去确定程序变量的类型
  • length的类型可以写成$\forall \alpha.list(\alpha) → integer $

多态语言的一个实例

  • 笛卡尔成绩表示函数的变量数量不做定义

  • dom（a) 表示的是声明在环境里的所有的变量
  • fresh ：对于所有的类型表达式，把所有的α都换成αi
  • Funcall函数调用，将函数的返回值T2，调用的T3和T1进行合一代换，结果是对应的T2的代换

代换、实例、合一

代换：类型表达式中的类型变量用其所代表的类型表达式去替换

把subst函数用于t后所得的类型表达式是t的一个实例，用S(t)表示

如果存在某个代换S使得S(t1) = S(t2)， 那么这两个表达式 t1和 t2 能够合一

第六章 运行时存储空间的组织和管理

概览

一些术语

第六章讨论的就是活动记录中的数据布局，以及执行函数的时候，活动记录的组织方式

术语 解释 备注
过程（函数）的活动 过程的执行 需要执行的代码，需要存储空间（局部变量，返回值，寄存器）
活动记录 可执行代码和存放所需信息的存储空间
生存期 过程体从开始执行到结束的时间
作用域（名字） 声明起作用的程序部分 即使一个名字在程序中只声明一次，该名字在程序运行时也可能表示不同的数据对象 （a++）
环境 把名字映射到左值（存储单元） 左值可以看成地址（比如变量，可更改），右值可以看成数据（可读），环境把名字绑定到内存单元
状态 把左值映射到右值（具体的值） 赋值语句，右值将数据写入内存单元，状态为赋值

过程可以简单看成是函数，具体可以看pascal的定义

局部存储分配 ：类似函数调用

静态概念和动态概念的对应

静态概念 动态对应 说明
过程的定义 过程的活动 类似于函数的定义和调用
名字的声明 名字的绑定 名字声明，动态被绑定到具体内存单元
声明的作用域 绑定的生存期 什么时候这个名字不再和内存绑定

活动记录：可执行代码和存放所需信息的存储空间。这是一段连续的存储区，可能会用寄存器顶替其中的域，每次过程调用的时候都会分配。

名称 作用 说明
临时数据 未声明的临时值 可能寄存器放不下了临时放一下计算结果
局部数据 比如变量 局部变量按照次序在局部数据域中分配空间
机器状态 调用前寄存器状态，比如pc作为返回地址等
访问链 非局部数据的访问
控制链 函数返回的位置，栈，寄存器数据等 用于指向调用者的活动记录

需要声明的可能已经知道的一些事实

  • 字节是可编址内存的最小单位
  • 存储空间可以根据其类型而静态确定
  • 局部数据的地址可以用相对于活动记录中某个位置的地址来表示
  • 数据对象的存储布局还有一个对齐问题（比如long 必须要4n开头）
  • 活动记录的位置是不定的，可以选择基准

程序块：类似于函数的大括号（本身有局部变量声明），允许嵌套，**按照最接近的作用域规则（变量）**即局部优先级大于整体，并列的程序块并不会同时活跃，变量可重叠分配（并列非嵌套，一定是一个结束另外一个开始，使用同一块内存就可以，不会重叠）
