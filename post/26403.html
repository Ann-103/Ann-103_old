hexo 建站记录

发表于2022-10-14|更新于2022-11-06|blog
|字数总计:23.3k|阅读时长:88分钟|阅读量:

本篇日志会记录对博客搭建所进行的美化和改动，并不是教程，部分内容会省略。

ps: 加减号分别代表删去和增添，出现问题先检查空格

为了区分，以后配置文件或者主题配置文件均指butterflyconfig，hexo的根目录配置文件是config.yml会强调名字

博客搭建

前置内容

  • node.js 下载

  • git 安装与配置

    备忘：ssh绑定了我a邮箱

快速开始

  1. github注册并新建仓库，命名为<用户名>.github.io，新建文件index.html并提交

  2. 生成ssh key，解决本地和服务器的链接问题，首先打开电脑文件夹，找到C:\Users\名字.ssh文件夹并删除，git bash:

    输入三次回车

    打开.ssh文件夹，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容

  3. 打开github主页，进入setting-> access->ssh and gpg keys，把复制的内容粘贴进去，自己命名保存

    输入ssh -T git@github.com验证，如果崩出来个选项输入yes回车，如果返回Hi！ ，You’ve…即成功。

  4. 新建文件夹（后面称这个位置为根目录），右键 Git Bash Here

  5. 输入hexo init

  6. 打开_config.yml 文件，搜索关键词deploy修改：

  7. 安装自动部署发布工具npm install hexo-deployer-git --save

  8. 执行对网页进行编辑和应用的三个命令：

    这里一直报错spawn faild来着，原本以为是网络问题，https://1187100546.github.io/2019/11/24/spawn-failed/ ，感谢这个解决方法，如果有相同报错可以尝试，但首先要检查空格

  9. 通过 <name>.github.io 访问你的网页

  10. 在之后的修改中，先不提交，但想测试效果可在hexo g后使用hexo s命令，访问：http://localhost:4000/ 查看

  11. 新建博客的命令

  12. 安装butterfly 主题

  13. 修改_config.yml theme:butterfly

  14. 安装pug和stylus的渲染器：

  15. 为了防止升级的时候有麻烦，在根目录新建文件_config.butterfly.yml，并把主題目录的 _config.yml 內容复制到 _config.butterfly.yml （且优先级较高）

  16. 绑定域名（可选）：https://www.likecs.com/show-30474.html

初始化项目后根目录文件夹的内容：

【node_modules】：依赖包
【scaffolds】：生成文章的一些模板
【source】：用来存放你的文章
【themes】：主题
【.npmignore】：发布时忽略的文件（可忽略）
【_config.landscape.yml】：主题的配置文件
【_config.yml】：博客的配置文件
【package.json】：项目名称、描述、版本、运行和开发等信息

主页文章节选(自动节选和文章页description)

在butterfly里，有四种可供选择

  • description： 只显示description，麻烦，要自己添加描述
  • both： 优先选择description，如果没有配置description，则显示自动节选的内容
  • auto_excerpt：只显示自动节选（字数限制）
  • false： 不显示文章内容
