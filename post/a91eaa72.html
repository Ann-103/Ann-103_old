Git学习笔记

发表于2022-10-23|更新于2022-11-06|tool
|字数总计:1.7k|阅读时长:5分钟|阅读量:

本文是针对git的工作原理和操作为主的学习记录,参考资料：https://missing.csail.mit.edu/2020/version-control/，《github入门与实践》，参考网站：https://www.liaoxuefeng.com/wiki/896043488029600/900003767775424

Git 代码的托管平台

Git是一个开源的分布式版本控制系统，在多人合作同一个项目，或者对于项目备份和版本回溯有要求的，可以使用git 将源码托管到代码托管平台上（Github，Gitee,Gitlab）。

将源码克隆到本地之后，可以自己进行改动，通过Pull Request功能申请合并，从而对代码进行修改

分布式控制系统是相对于集中式而言的，集中式将所有的数据都放在服务器里，开发者需要checkout后进行修改再commit，即保证能够连接上服务器。

Git

安装

在 Windows 环境中，最简单快捷的方法是使用 msysGit A。按照 Downloads 的向导下载安装包。然后选择组件、设置环境变量等。Git Bash 会作为应用程序添加进系统，有关git的命令我们只在这个窗口进行。

建议搜索将gitbash集成在windows终端的方法，美观且方便复制粘贴

初始化

  • 设置git时候的姓名和邮箱地址

    命令生成~/.gitconfig文件，可以设置

    从而让命令的输出有更高可读性

  • 本地库初始化 git init，选择了一个文件夹作为工作区

    另外由于linux系统中自动配置了git，如果在windows中使用wsl的话也是可以直接进行git的操作的

  • 初始化完成后，根目录会多出来一个.git的文件夹，称之为版本库，包含了暂存区，以及git自动创建的master分支，使用commit命令可以将文件添加到本地仓库中

    • 以及指向master的一个指针叫HEAD

  • 使用git clone 获取github上的项目仓库

命令操作

  1. 提交

    • 对本地文件进行修改，工作区文件添加到暂存区，再提交

    • 提交到本地仓库

  2. 回退

    • 查询log，然后看需要回退多少步

      因为Git在内部有个指向当前版本的HEAD指针，相当于指针的更改

  3. 编辑

    • 暂存区文件删除

      暂存区文件删除，同时删除工作区文件

      取消暂存

    • 未上传时撤销对文件的更改，回到最近一次git commit或git add时的状态

      同时，如果误删了工作区的文件，使用这个操作可以直接把某个文件恢复，前提是版本库中有这个文件的其他版本

    • 隐藏当前变更，以便能够切换分支

    • 重命名文件，并将已改名文件提交到暂存区

  4. 查询

    • git status 查看当前分支状态，甚至可以提示新增的文件如何删去等操作

    • 工作区中，已修改未提交的文件对比

    • 比较仓库和暂存区的不同

    • 比较暂存区与上一版本的差异

    • 查看历史记录：从近到远

      commit id每个人都不一样，加密后是sha数字

    • 记录每一次命令，用来查找被撤销的新操作id等

远程仓库（重点）

本地仓库使用git commit更改版本，远程仓库使用git pull，一般会设置权限，想推也推不了

初始机器有原始版本库（充当服务器），不同机器进程clone，各自pull request 到服务器仓库里，也从服务器仓库中拉取别人的提交。使用github完成这个功能

建立自己的电脑与github 的密钥连接，获取主机sshkey然后在github上添加即可

个人仓库

  • 申请github账号并创建repo仓库，将自己本地的仓库与之关联，本地运行

    只有自己的库才有推送的权限，即电脑的ssh key被github账号认证

  • 把本地库的所有内容推送到远程库上

    由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git还会把本地的master分支和远程的master分支关联起来，以后不用

  • 解绑

克隆库

从github克隆一个本地库，多人协作开发则人均克隆一份即可

  • 分支：不是开叉哦，是时间上的并行，对其他分支透明的进度更改，可以将两个分支合并

    • 首先，我们创建dev分支，然后切换到dev分支

    • 查看当前分支

    • 在分支上提交

    • 合并指定分支到当前分支，直接把master指向dev的当前提交

多人协作

  • 要查看远程库的信息，用git remote -v，如果没有推送权限，就看不到push的地址。

  • 推送分支，就是把该分支上的所有本地提交推送到远程库

  • 冲突：另外一个人在克隆后，创建本地分支

    先用git pull把最新的提交从origin/dev抓下来，然后，在本地合并，解决冲突，再推送
